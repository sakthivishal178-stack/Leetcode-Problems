# Intuition
The versions are sorted in a way that all versions before the first bad version are good, and all versions after it are bad. This monotonic property allows us to use Binary Search to efficiently find the first bad version while nimizing API calls.

# Approach
- Initialize two pointers:
  - `low = 1`
  - `high = n`
- While `low < high`:
  - Compute `mid = (low + high) // 2`.
  - If `isBadVersion(mid)` is `True`, the first bad version is at `mid` or before it, so set `high = mid`.
  - Otherwise, the first bad version must be after `mid`, so set `low = mid + 1`.
- When the loop ends, `low` (or `high`) points to the first bad version.

# Complexity
- Time complexity:
  - **O(log n)**

- Space complexity:
  - **O(1)**

# Code
```python
# The isBadVersion API is already defined for you.
# def isBadVersion(version: int) -> bool:

class Solution:
    def firstBadVersion(self, n: int) -> int:
        low = 1
        high = n

        while low < high:
            mid = (low + high) // 2

            if isBadVersion(mid):
                high = mid
            else:
                low = mid + 1

        return low
