# Intuition
We need to count all contiguous subarrays whose product is less than `k`.

Instead of checking every possible subarray (which is too slow), use a **sliding window**. Expand the window by moving the right pointer and keep multiplying the elements. Whenever the product becomes greater than or equal to `k`, shrink the window from the left until the product becomes valid again.

For every valid window ending at `right`, all subarrays ending at `right` are also valid, so we add `(right - left + 1)` to the answer.

# Approach
1. Handle the edge case where `k <= 1`, since no positive product can be smaller than `1`.
2. Initialize:
   - `left = 0`
   - `product = 1`
   - `count = 0`
3. Iterate through the array with the `right` pointer:
   - Multiply the current element into `product`.
   - While `product >= k`, divide out the leftmost element and move `left` forward.
   - After the window becomes valid, add `(right - left + 1)` to the answer because every subarray ending at `right` within the current window has a product less than `k`.
4. Return `count`.

# Complexity
- **Time complexity:** `O(n)`
  - Each element enters and leaves the sliding window at most once.

- **Space complexity:** `O(1)`
  - Only a few extra variables are used.

# Code
```python
class Solution:
    def numSubarrayProductLessThanK(self, nums: List[int], k: int) -> int:
        if k <= 1:
            return 0

        left = 0
        product = 1
        count = 0

        for right in range(len(nums)):
            product *= nums[right]

            while product >= k:
                product //= nums[left]
                left += 1

            count += (right - left + 1)

        return count
